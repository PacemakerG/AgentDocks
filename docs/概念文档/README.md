# 概念文档写作规范

本目录存放学习过程中的基础概念笔记，供后续复习查阅。新增文档请遵循以下风格。

## 文件命名

`主题名.md`，主题可带副标题，例如 `Go语言基础概念（对比C++与Python）.md`。

## 文档结构

- 一级标题为文档主题，与文件名一致。
- 正文按知识点拆分为多个二级标题（`## 1. xxx`、`## 2. xxx`），编号递增。
- 每个二级标题之间用 `---` 分隔。
- 最后一节通常是"整体关系"或"核心概念对照表"，做全文小结。

## 内容风格

- 尽量简洁，不写背景铺垫、不写总结性废话，直接给知识点和示例。
- 优先用代码块展示流程或语法，而不是大段文字描述：
  - 流程用 ` ```text ` 代码块画箭头图（`→`）。
  - 语法/命令用对应语言的代码块（` ```go `、` ```bash ` 等）。
- 有对比学习需求时（如 Go 对比 C++/Python），每个知识点后紧跟对应语言的等价写法。
- 用简短要点列表（`-`）代替长句子。
- 全文结尾可用表格做多语言/多概念对照，一目了然。

## 不要做的事

- 不加入与知识点无关的引言、过渡句、鼓励性总结。
- 不重复解释已经在代码示例中体现的内容。
- 不使用夸张的强调（如大量加粗、感叹号）。
