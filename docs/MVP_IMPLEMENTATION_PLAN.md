# AgentDock MVP 实现计划

> 目标周期：20～30 天
>
> 核心目标：跑通“可执行、可观测、可评测、可回放、可改进”的最小闭环。

## 一、这个月要交付什么

第一个月不追求做成完整平台，只交付一条真实可运行的链路：

```text
从离线评测集选择任务
    ↓
AgentDock 创建独立运行环境
    ↓
Coding Agent 执行任务
    ↓
记录执行过程、文件变化和结果
    ↓
CCWhat 展示、回放并评分
    ↓
修改 Prompt 或配置后重新执行
    ↓
对比两次结果
```

完成后应当能演示：选择一个评测任务，用一条命令运行 Agent，在 CCWhat 中看到完整过程和评分；修改 Agent 后，再次执行同一任务并比较结果。

## 二、MVP 的明确边界

### 必须完成

- 在独立 Workspace 中运行一个 Coding Agent。
- 支持正常结束、超时和主动取消。
- 保存标准输出、错误输出、文件变化和 Git Diff。
- 每次执行生成唯一编号，并关联评测任务、Agent 和运行环境。
- CCWhat 能读取一次执行的完整记录。
- 离线评测集能对结果评分。
- 能使用相同输入重新执行，并对比两次结果。
- 提供最小的环境隔离和资源限制，失控任务不能影响其他任务。
- 提供自动测试、演示脚本和使用说明。

### 暂时不做

- 多机器调度。
- Kubernetes 和 MicroVM。
- 多 Agent DAG。
- 自动修改 Prompt 或自动优化 Agent。
- 复杂的用户、租户和计费系统。
- 自己从零实现底层沙箱技术。

第一版的“可改进”采用人工闭环：CCWhat 给出证据和评分，人根据结果调整 Prompt、工具或配置，然后重新运行。

## 三、如何借鉴现有项目

不要 Fork 一个大型项目后直接修改。AgentDock 保留自己的协议和主流程，只学习或复用成熟项目中已经解决好的部分。

### 开始前重点阅读

1. **Anthropic Sandbox Runtime**
   - 只看权限如何配置、默认禁止什么、违规时如何报告。
   - 输出一页笔记：AgentDock 第一版允许什么、禁止什么。

2. **SWE-ReX**
   - 只看一个任务如何提交、命令如何执行、日志如何返回。
   - 输出一页笔记：AgentDock 的输入、输出和运行状态。

3. **Sandlock**
   - 第一遍只看整体结构和最小示例。
   - 等需要加强隔离时，再看它如何限制文件和进程。

4. **OpenAI Codex**
   - 只看命令执行、取消、事件记录相关模块。
   - 学习成熟 Rust 项目的代码组织，不尝试完整阅读。

### 借鉴原则

- 优先借鉴接口、测试方法和安全边界。
- 优先调用成熟工具，不从零实现底层隔离。
- 必须复用代码时，先确认许可证并记录来源。
- 在文档中写清楚：借鉴了什么、自己实现了什么、为什么修改。

## 四、技术安排

### Rust 怎么学

不要先花一个月系统学习 Rust，也不要完全不学就直接写项目。

前 5～7 天只学习项目马上会用到的内容：

- 变量、结构体和枚举。
- 错误处理。
- 文件和 JSON 读写。
- 启动外部命令。
- 异步读取日志。
- 任务取消和超时。
- 基础测试。

学习方式是“学一个知识点，马上在 AgentDock 中使用”。暂时不学习与 MVP 无关的高级语法。

### 语言分工

- **Rust**：实现 AgentDock 的本地执行程序。
- **Python**：连接现有评测集、准备任务和调用评测逻辑。
- **CCWhat 原有技术栈**：负责展示、回放、诊断和结果对比。

第一个月不引入 Go。等单机闭环稳定后，再为完整控制面学习 Go。

## 五、20～30 天行动计划

### 第 1～3 天：固定闭环和数据格式

#### 要做的事

- 从离线评测集中选择 3～5 个代表性任务。
- 手工跑通其中一个任务，确认输入、代码仓库和评分方式都可用。
- 定义一次运行需要保存的信息：
  - 评测任务编号。
  - Agent 名称和配置。
  - 代码版本。
  - 执行命令和环境配置。
  - 开始时间、结束时间和最终状态。
  - 日志、文件变化、评分和产物位置。
- 定义最少的运行状态：等待、运行、成功、失败、取消、超时。
- 画出 AgentDock 与 CCWhat 之间的数据流。

#### 验收

- 一个评测任务可以用固定格式描述。
- 一次运行的输入和输出能够保存到一个目录中。
- CCWhat 知道需要读取哪些内容。

### 第 4～7 天：Rust 入门和最小执行程序

#### 要做的事

- 完成 Rust 基础学习，只覆盖本项目需要的内容。
- 创建最小的 `agentdock run` 命令。
- 接收 Workspace、执行命令和超时时间。
- 启动普通命令，实时返回输出。
- 保存退出状态和运行时间。
- 为正常执行和命令失败编写测试。

#### 验收

下面的流程可以稳定运行：

```bash
agentdock run --workspace ./repo --timeout 60 -- command args
```

执行结束后能看到日志、退出状态和运行时间。

### 第 8～12 天：独立环境和执行证据

#### 要做的事

- 每次任务创建独立 Workspace，不直接修改原仓库。
- 接入一种成熟的隔离方式，第一版只封装和管理它。
- 支持超时和主动取消。
- 确保任务结束后没有残留进程。
- 保存任务前后的文件列表和 Git Diff。
- 限制最大运行时间、最大日志量和基本资源使用。
- 将执行过程写成逐行事件记录，方便 CCWhat 读取。

#### 验收

- 两个任务可以同时运行，彼此文件不受影响。
- 超时或取消后没有残留进程。
- 原仓库不会被任务直接修改。
- 可以得到完整日志、最终状态和 Git Diff。

### 第 13～17 天：接入 CCWhat

#### 要做的事

- 在 AgentDock 和 CCWhat 中统一任务编号和运行编号。
- 让 CCWhat 导入 AgentDock 生成的事件记录。
- 展示一次运行的时间线、日志、文件变化和退出原因。
- 关联原始评测任务与最终评分。
- 对异常或缺失数据给出明确提示。

#### 验收

- 在 CCWhat 中选择一次运行，可以看到从开始到结束的完整过程。
- 能从评测任务跳转到对应运行记录。
- 失败、取消和超时可以被正确区分。

### 第 18～21 天：评测、回放和对比

#### 要做的事

- 运行离线评测逻辑并保存评分。
- 保存本次运行使用的代码版本、Agent 配置和环境信息。
- 实现“重新运行同一任务”。
- 在 CCWhat 中对比两次运行：
  - 是否成功。
  - 最终评分。
  - 执行时间。
  - 文件变化。
  - 失败原因。
- 手工完成一次“发现问题—修改配置—重新运行—结果变好”的演示。

#### 验收

- 同一评测任务可以稳定重新执行。
- 两次运行的输入差异和结果差异清晰可见。
- 可以用一个真实案例说明系统如何帮助改进 Agent。

完成到这里，MVP 闭环已经成立。如果时间只有 20 天，应优先保证这里稳定可演示。

### 第 22～25 天：安全和失败测试

#### 要做的事

- 测试任务读取 Workspace 外文件时是否会被阻止。
- 测试关闭网络和允许指定网络目标。
- 测试无限输出、长时间运行和大量子进程。
- 测试 AgentDock 意外关闭后的任务状态是否清楚。
- 为每种失败准备固定测试任务。
- 修复测试发现的问题，不增加新的功能范围。

#### 验收

- 常见失控任务不会拖垮宿主环境。
- 每种失败都有清楚的结束原因和执行记录。
- 测试可以重复运行，而不是只手工验证一次。

### 第 26～30 天：整理成可展示项目

#### 要做的事

- 编写五分钟 Quick Start。
- 准备一条命令启动完整演示。
- 在自动检查中运行格式检查、单元测试和基础安全测试。
- 使用 5～10 个评测任务运行完整流程。
- 记录执行成功率、平均时间和失败类型。
- 编写架构说明、安全边界和已知限制。
- 录制一次完整演示视频或准备演示脚本。

#### 验收

- 新用户按照 README 可以独立跑通一个任务。
- 自动测试全部通过。
- 演示不依赖临时手工修改。
- 文档明确说明哪些能力已经完成、哪些仍是后续计划。

## 六、最终验收清单

以下项目全部完成，才算 MVP 完成：

- [ ] 一条命令运行一个离线评测任务。
- [ ] Agent 在独立 Workspace 和受限环境中执行。
- [ ] 支持日志流、超时、取消和完整进程回收。
- [ ] 保存 Git Diff、运行配置、结果和评分。
- [ ] CCWhat 可以展示、回放和诊断一次运行。
- [ ] 同一任务可以重新运行并比较结果。
- [ ] 至少有一个“分析后改进成功”的真实案例。
- [ ] 正常、失败、超时、取消和越权场景都有自动测试。
- [ ] 提供 Quick Start、架构说明和演示材料。

## 七、进度取舍原则

如果时间不足，严格按照下面的优先级取舍：

```text
真实跑通闭环
> 运行结果完整可信
> 失败可以回收
> 相同任务可以重跑
> CCWhat 展示和评测
> 更强的隔离
> 更多 Agent 和更多运行后端
```

不要为了展示复杂架构，牺牲最小闭环的稳定性。
